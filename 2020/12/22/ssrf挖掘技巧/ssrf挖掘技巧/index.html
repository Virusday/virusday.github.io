0x02 简介

SSRF(Server-Side Request Forgery:服务器端请求伪造) 是一种由攻击者构造形成由服务端发起请求的一个安全漏洞。一般情况下，SSRF攻击的目标是从外网无法访问的内部系统。（正是因为它是由服务端发起的，所以它能够请求到与它相连而与外网隔离的内部系统）

SSRF漏洞形成的原因主要是服务器端所提供的接口中包含了所要请求的内容的URL参数，并且未对客户端所传输过来的URL参数进行过滤。这个漏洞造成的危害有：

1：可以对外网、服务器所在内网、本地进行端口扫描，获取一些服务的banner信息;
2：攻击运行在内网或本地的应用程序（比如溢出）;
3：对内网web应用进行指纹识别，通过访问默认文件实现;
4：攻击内外网的web应用，主要是使用get参数就可以实现的攻击（比如struts2，sqli等）（毕竟外网访问不到或者受限制，只能伪造服务器的请求）;
5：利用file协议读取本地文件等。

0x03 可能存在的坑

  • 测试支持的协议:http、https、gopher、telnet 等协议。
  • SSRF 的类型为无状态型 SSRF ，即不管你请求的端口是否开放、协议是否支持、网站是否能访问 返回的状态都是一样的，你无法通过它扫描端口开放情况和使用 file 协议读取本地信息。
  • 不支持 302 跳转

0x04 挖掘+技巧

挖掘漏洞：

对外发起网络请求的地方都可能存在SSRF漏洞，列举
图片加载下载，分享页面，在线翻译，未公开的api（从远程服务器请求资源
文件处理，编码处理, 属性信息处理。

挖掘方式：

分为两大类：1、web功能点 2、数据包传参

利用＋绕过

1、后缀名检测

输入http://ceye.io却发现并没有回显请求，这是因为这种地方一般会对后缀做一个检测，如果不是jpg或者其他图片后缀的话并不会通过，不过好在http://ceye.io是很强大的，我们可以随便构造，http://abcdef.ceye.io/ichunqiu.jpg, 对其进行输入，再去观察有无回显。

绕过

  • @
  • 添加端口号
  • 短网址绕过
  • 指向任意ip的域名绕过 xip.io
  • p限制绕过的话就利用进制转换（十进制、八进制、十六进制这些）
  • 协议绕过（只支持http(s)的话可以构造302跳转再转换协议、后面有提到。）
  • DNS rebinding

dns重绑定可以利用与ssrf绕过、bypass同源策略等

1、 特定域名ttl=0

2、绑定两条A记录，一条内网、一条为外网，第一次解析成外网，第二次解析到内网就可以成功。（但是dns有缓存机制？）

0x05 找到ssrf漏洞之后

1、 PHP

​ 找到ssrf漏洞之后，应该去检查这几件事，试试从低位拉到高位

  1. 摸清SSRF的状态是否为布尔状态（看返回包返回的信息，也有可能无回显盲注去打打）题外：url跳转测试以下CRLF漏洞Set-Cookie以下

  2. 此SSRF支持什么协议（gopher、http、telnet、dict）（1、4扫描redis写webshell）

  3. 此SSRF是否支持302跳转（如果限制ip端口这些类型访问啥的，如果支持跳转可利用的面更广一点）（外连端口用nc确定ip地址）

得出结论：

此SSRF为布尔型SSRF，能探测http应用并获取html源码、支持302跳转，可尝试通过响应速度来探测端口但误报率很高（某些端口可能是一直连接，最后超时）

2、 Java

1. 利用方式
网络请求支持的协议如下
http，https，file，ftp，mailto，jar，netdoc
对比php的ssrf，java这块利用相对局限，

2. 总结
以下几种类引用不当会造成SSRF
Request类，URL类的openStream，HttpClient类，URLConnection和HttpURLConnection类，
修复方案：
• 限制协议为HTTP、HTTPS
• 不用限制302重定向
• 设置URL白名单或者限制内网IP
